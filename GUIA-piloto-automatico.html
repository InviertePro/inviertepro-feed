Tu Instagram en piloto automático

Esta guía deja Instagram publicando solo, vía la API oficial de Meta. El setup se hace una vez; después, cada día se publica la próxima pieza sin que muevas un dedo. Sigue los pasos en orden.

🟠 Lo haces tú

Todo lo que sea entrar a tu Instagram y a Meta (cuenta, app, permisos, token). Nadie más puede: es tu identidad.

🔵 Lo hago yo

El código, el manifiesto con tus 20 piezas y copys, y dejar el publicador andando. Ya están listos.

Cómo funciona (30 segundos de teoría)

La API de Instagram publica al instante cuando se la llama — no "agenda" a futuro. Así que el piloto automático se arma así: dejamos una tarea diaria que ejecuta el script una vez por día, y el script toma la próxima pieza pendiente del manifiesto y la publica. Simple y robusto. Para que la API pueda usar las imágenes, tienen que estar en una URL pública (Instagram las descarga por internet), por eso hay un paso de "hosting".

1

Cuenta de Instagram Profesional + página de Facebook Tú ~5 min

La API solo funciona con cuentas Business o Creator vinculadas a una página de Facebook. Es gratis.

  1. En la app de Instagram: Configuración → Tipo de cuenta y herramientas → Cambiar a cuenta profesional → Empresa.
  2. Crea (o vincula) una página de Facebook de InviertePro. Si no tienes, se crea en 2 minutos en facebook.com.
  3. Enlaza la cuenta de Instagram con esa página (Instagram te lo ofrece en el mismo flujo, o desde la página de FB → Configuración → Cuentas vinculadas).
2

Crear la app en Meta for Developers Tú ~10 min

Es la "llave" técnica que permite publicar. También gratis.

  1. Entra a developers.facebook.com e inicia sesión con tu Facebook. Acepta registrarte como desarrollador (es automático).
  2. Mis Apps → Crear app. Tipo: Empresa / Business. Ponle nombre, ej. "InviertePro Publisher".
  3. Dentro de la app, agrega el producto Instagram Graph API (o "Instagram" → API con inicio de sesión de Facebook).
  4. En Configuración → Básica, anota el App ID y el App Secret (los necesitarás para el token largo).
Como vas a publicar en tu propia cuenta, no necesitas la "Revisión de la app" de Meta: basta con que tu usuario tenga rol de administrador en la app (modo Desarrollo). La revisión formal solo se exige para publicar en cuentas de terceros.
3

Generar el token de acceso Tú ~10 min

El token es la contraseña que autoriza a publicar. Vamos por uno de larga duración.

  1. Abre el Graph API Explorer: developers.facebook.com/tools/explorer.
  2. Arriba a la derecha, elige tu app. Haz clic en Generar token de acceso y otorga estos permisos:
    instagram_basic · instagram_content_publish · pages_show_list · pages_read_engagement · business_management
  3. Copia ese token (dura ~1 hora). Conviértelo a uno de 60 días pegando esto en el navegador (reemplaza APP_ID, APP_SECRET y TOKEN_CORTO):
    https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=APP_ID&client_secret=APP_SECRET&fb_exchange_token=TOKEN_CORTO
    Te devuelve el token largo. Ese es el que usaremos.
🔒 El token es sensible como una contraseña. No lo publiques en ningún lado público. Para que no expire nunca (ideal para piloto automático), más adelante conviene un usuario de sistema en la Configuración del Negocio de Meta — te ayudo con eso cuando lleguemos ahí.
4

Obtener tu IG User ID Tú ~2 min

Es el número de tu cuenta de Instagram profesional. En el mismo Graph API Explorer, corre estas dos consultas:

  1. me/accounts → te da el ID de tu página de Facebook.
  2. {ID_PAGINA}?fields=instagram_business_account → te devuelve el instagram_business_account.id. Ese número es tu IG User ID.
5

Hospedar las imágenes en una URL pública Tú te guío yo

La API descarga cada imagen por internet, así que la carpeta de PNG debe estar en una dirección pública. Elige una:

RECOMENDADO

Tu dominio inviertepro.cl

Sube la carpeta por tu hosting (FTP/panel). Quedan URLs limpias tipo inviertepro.cl/feed/C-001/C-001_01.png.

Repositorio público en GitHub

Gratis y estable. Subes la carpeta y usas las URLs raw.githubusercontent.com/.... Te lo dejo armado si quieres.

Cloudinary / Amazon S3

Servicios de almacenamiento con capa gratis. Buenos si además vas a hospedar los videos de los reels.

¿No quieres complicarte?

Dímelo y te propongo la más simple según tu hosting actual. Lo importante: que las URLs sean públicas y directas.

Cuando definas el host, editas una línea en manifest.json: el campo base_url. Todo lo demás ya está mapeado ({base_url}/C-001/C-001_01.png, etc.).
6

Configurar y probar el publicador Yo con tus datos

El script publicar_instagram.py y el manifest.json ya están hechos. Solo falta enchufar tus credenciales:

# opción rápida: crea config.json (copia config.example.json)
{ "ig_user_id": "TU_IG_USER_ID", "token": "TU_TOKEN_LARGO" }

# 1) ver estado
python3 publicar_instagram.py --status

# 2) simular la próxima (no publica nada)
python3 publicar_instagram.py --dry-run --next

# 3) publicar de verdad la próxima pieza lista
python3 publicar_instagram.py --next

Empieza publicando una con --next para ver que todo calza en tu feed. Si te gusta, seguimos.

7

Dejarlo en piloto automático Yo último paso

Con el publicador andando, programamos una tarea diaria (días hábiles) que corra --next. Así se publica una pieza por día, sola, hasta vaciar la cola. Cuando tengas los videos de los reels, los sumamos al manifiesto y siguen igual de solos.

Detalle importante para que "no pare nunca": el token de 60 días hay que renovarlo, o mejor, cambiarlo por uno de usuario de sistema que no expira. Lo dejamos configurado en este paso para que el piloto automático sea de verdad permanente.

✅ Lo que necesito de ti para terminar

Tu IG User ID (el número del paso 4)
El token largo (paso 3) — es sensible; me lo pasas por acá y lo uso solo para configurar, o lo corres tú mismo con la guía.
Dónde hospedarás las imágenes (paso 5) — tu dominio, GitHub u otro. Con eso ajusto el base_url.
Los videos de los reels (opcional, después) — mientras, publicamos los 14 carruseles y datos, que ya están 100% listos.
